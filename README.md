# Project Windos (x86-64)

> **一個為了挑戰現代硬體驅動限制而生的 FreeDOS 擴展核心計畫。**

** 專案背景
在開發 FreeDOS 的過程中，我發現許多現代硬體（尤其是 x86-64 架構）因為缺乏驅動程式而無法啟動。這個計畫的初衷是透過手寫組合語言（Assembly），嘗試在底層直接控制硬體，並最終實現一個相容於 FreeDOS 環境但能發揮現代 CPU 效能的核心。

本專案深受 **TempleOS** 與《內嵌作業系統》一書的啟發。

** 技術規格
- **架構**: x86 (Real Mode 16-bit) -> 邁向 x86-64 (Long Mode)
- **開發語言**: NASM Assembly
- **執行環境**: FreeDOS / 實機 (CHIMEI Desktop) / 虛擬機
- **開發工具**: Volkov Commander (VC) 內建編輯器

**目前進度: v0.25 ALPHA
- [x] 環境初始化 (BIOS 80x25 文字模式)
- [x] 基礎系統資訊視窗顯示
- [x] 使用者互動 (鍵盤監聽)
- [x] 硬體級別重啟功能 (`JMP 0FFFFh:0000h`)
- [ ] 進入保護模式 (Protected Mode)
- [ ] 64 位元長模式 (Long Mode) 切換

** 檔案說明

- `.gitignore`: 自動過濾編譯產生的 `.com` 與 `.obj` 檔案。
